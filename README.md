# Avito Tech — Test Assignment

Тестовое задание: приложение для управления объявлениями с каталогом, просмотром карточки товара, страницей редактирования и AI-подсказкой рыночной цены.

## Что реализовано

Проект включает базовый набор сценариев для работы с объявлениями:

- каталог объявлений с пагинацией
- фильтрация и сортировка
- страница отдельного объявления
- форма редактирования
- валидация обязательных полей
- AI-подсказка рыночной цены
- уведомления об успешном и неуспешном сохранении

## Установка и запуск

### Клонирование репозитория

```bash
git clone <repo-url>
cd project
```

### Установка зависимостей и запуск
Из корневой папки проекта:

```bash
npm install
npm run install:all
npm run dev
```

После этого одновременно запустятся frontend и backend.

## Особенности реализации

### UI без готовых библиотек

Интерфейс собран вручную, без использования прочих UI-библиотек.  
Такой подход позволил лучше контролировать структуру компонентов, стили и адаптацию интерфейса под предоставленный макет.

### Логика формы

Страница редактирования реализована путем написания логики формы вручную, включая:

- хранение состояния полей
- отслеживание изменений формы
- валидацию `title` и `price`
- `touched`-состояние полей
- отдельное хранение ошибок

### Redux Toolkit

Redux Toolkit используется для управления состоянием каталога: фильтрацией, сортировкой и списком объявлений.

Это был мой первый опыт работы с Redux в проекте, поэтому часть подходов и решений я разрабатывал с помощью нейросетей.

### AI-интеграция

На сервере реализованы endpoint'ы:

```http
POST /ai/price-suggestion
```

Он анализирует данные объявления, предлагает ориентировочную рыночную цену и возвращает краткий комментарий.

```http
POST /ai/description-suggestion
```

Он также анализирует данные объявления, и предлагает или улучшает и возвращает краткое описание.

Используемая модель: `gpt-4o-mini`.

Важно: API-ключ хранится только на сервере и не передаётся на клиент. После копирования в папке server будет файл .env.example, где лежит поле OPENAI_API_KEY.
Чтобы проверить логику, могу предоставить ключ (нужно будет переименовать файд в .env), также приложу видео с демострацией.

### Типизация

В проекте активно используется TypeScript.

Все основные типы вынесены в отдельный файл `types.ts`, включая:

- `Item` — модель объявления
- `FormDataType` — данные формы редактирования
- `ItemCategory` — категории объявлений
- `CategoryFieldConfig` — конфигурация полей для динамической формы

### Адаптация к пользовательским настройкам (rem)

В проекте используется собственный SCSS-подход с функцией `rem()` для задания размеров.

Это позволяет интерфейсу корректно масштабироваться при изменении настроек браузера (например, увеличении размера шрифта).

Пример использования:

```scss
font-size: rem(16);
```

https://github.com/user-attachments/assets/e6f8100d-fef3-4d98-8e42-79434dc66c82

На видео показано, как интерфейс реагирует на изменение масштаба текста в браузере.
